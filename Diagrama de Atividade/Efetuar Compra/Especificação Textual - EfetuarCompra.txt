Caso de Uso: Efetuar Compra

Ator principal: Administrador

Pré-condições: Caixa aberto.

Fluxo Principal (cenário de sucesso):

Este caso de uso inicia quando o administrador acessa o sistema desktop para realizar uma compra do fornecedor.
O sistema verifica se o caixa está aberto.
O administrador seleciona o fornecedor.
O sistema pesquisa o fornecedor.
O administrador seleciona os produtos.
O sistema calcula o total da compra.
O administrador escolhe o método de pagamento.
O sistema finaliza a compra e gera o pedido.
Pós-condições: Pedido gerado e compra registrada.

Fluxos Alternativos:
2. O caixa está fechado.
  2a. O sistema cancela a operação.
3. O fornecedor não está cadastrado.
  3.1. O sistema emite mensagem "Fornecedor não cadastrado" e questiona se o administrador deseja pesquisar outro ou cadastrá-lo.
  3.2. Retornar ao passo 5 após o cadastro.
6. O administrador opta por cancelar a operação.
